# 面试问答准备

## 为什么选 DeepSeek，而不是 GPT？

项目使用 DeepSeek 的原因是中文业务文本处理表现合适、接入成本可控，并且提供 OpenAI 兼容接口，便于接入 Dify 与后续替换模型。方案层面保留模型可替换性，不依赖单一厂商。

## 评分权重怎么确定？

按岗位职责拆成技能匹配 40%、相关经验 25%、项目相关性 20%、综合质量 15%。先让技能和可迁移项目经验占主要权重，再用测试样例校验推荐结论；权重不是客观真理，应随业务反馈迭代。

## 为什么不让 AI 自动筛掉候选人？

招聘属于高影响场景。模型只能辅助整理、给出证据和待确认项，最终结论必须由招聘人员复核；信息缺失时流程输出 supplement，而不是自动淘汰。

## JSON 解析失败怎么办？

LLM 后加入 Code 节点：先清理可能的代码围栏和前后说明，再尝试 JSON 解析。失败时返回 parse_failed 和原始输出，确保流程不崩溃并支持人工排查。

## 最大挑战是什么？

第一是变量必须通过 Dify 选择器绑定，不能手写猜测节点路径；第二是模型可能输出 think 内容，因此通过 Code 节点提取 JSON，并在最终输出中隐藏原始 LLM 文本。

## 如果有更多时间怎么改进？

接入飞书多维表格形成审核闭环；增加岗位评分规则配置化；用更多脱敏样本评估一致性；对不同 Prompt 和模型进行 A/B 测试。
