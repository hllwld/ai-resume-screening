# 3 分钟 Demo 讲稿

## 0:00–0:25｜问题与边界

“这是一个 AI 简历批量评估工作台。它帮助招聘人员读取简历、对照岗位要求、
整理匹配证据和待确认项，但不会自动决定录用或淘汰，最终结论必须由人做出。”

## 0:25–0:55｜架构

“前端使用 Vue 3 和 PDF.js。PDF 原文件留在浏览器，只把提取后的文本发送给
FastAPI。后端隐藏 Dify API Key，负责登录鉴权、并发、每日限额和任务隔离，
再调用 Dify Workflow 与 DeepSeek 生成结构化结果。”

## 0:55–1:30｜批量输入

“登录后可以选择一个简历文件夹或多个 PDF，填写岗位 JD，并选填补充评价要求。
页面会在提交前检查文件大小、页数、文本长度、扫描件和加密文件。点击批量评估后，
可以看到总体进度以及每份简历的成功、失败或取消状态。”

## 1:30–2:15｜可解释结果

“结果表支持按候选人、推荐结论和分数筛选排序。展开详情后能看到技能、经验、
项目和综合质量四个维度，以及匹配证据、缺失信息、风险标记和建议追问。信息不足时，
系统要求补充材料或人工复核，不自行推断。”

## 2:15–2:40｜Excel 人工复核

“批次完成后可以直接下载格式化 Excel。工作簿包含筛选、条件格式、人工复核
下拉框、动态行高和失败处理说明，招聘人员可以在 AI 结果基础上补充自己的判断。”

## 2:40–3:00｜工程化与总结

“公网版本运行在腾讯云香港轻量服务器，通过 Caddy 自动提供 HTTPS。服务限制
单 IP 每天 10 份、全站每天 50 份，并对访问口令、任务和下载结果进行会话隔离。
这个项目展示了从 Prompt、Dify Workflow 和 JSON 容错，到前后端、部署和人工
复核交付的完整 AI 应用落地流程。”
